# 微信聊天记录批量抓取与整理工具

这是一个用于**批量抓取微信 PC 端聊天记录**并**自动整理排序、去重**的 Python 工具集。  
它可以帮助你快速导出大量微信聊天内容（包括历史消息），并生成按时间先后顺序排列、无冗余的文本文件，便于存档、分析或备份。

以往的方案多通过破解本地加密文件来提取数据，这不仅极易因微信更新而失效，还存在一个相当相当重要的问题——隐私泄露风险！

本项目另辟蹊径，直接在本地通过官方客户端获取已解密的数据，从根本上杜绝了被限制的风险。

整个过程无需联网，所有操作均在您的电脑上离线完成，确保了数据的绝对安全与私密。

我们坚信“你的数据才真正属于你”，让您能安心地将个人记忆用于AI训练等个性化用途。

目前仍存在一些细小问题，还有很多可优化的空间。

---

## 📌 项目背景

微信 PC 版本身不支持一次性导出完整聊天记录（尤其当消息量较大时，手动复制容易遗漏或重复）。  
本工具通过模拟鼠标拖拽选区 + 滚动翻页，实现**自动化、可中断、可继续**的抓取流程；随后通过解析脚本，将抓取到的原始内容去除分割标记、剔除重复行，并严格按照时间戳排序，最终输出干净的文本文件。

---

## 🚀 功能特点

- **自动抓取**：模拟鼠标拖拽选择聊天区域 → 点击“复制”按钮 → 自动翻页，循环直至手动停止。
- **全局热键停止**：任意时刻按下 `Ctrl + Backspace` 即可安全终止抓取（已抓取内容自动保存）。
- **智能去重排序**：将抓取到的原始文件（含分块标记）清洗、去重、按时间顺序重排。
- **轻量易用**：仅依赖 `pyautogui` 和 `pyperclip` 两个第三方库，代码简洁，可自由修改坐标适配不同屏幕。

## 📂 文件说明

| 文件名                 | 作用                                                                                     |
| ------------------- | -------------------------------------------------------------------------------------- |
| `wechat_capture.py` | **抓取脚本**：在微信聊天窗口中运行，自动拖拽、复制、翻页，生成带分块标记的原始 `.txt` 文件。                                   |

| `requirements.txt`  | 项目依赖清单（`pyautogui`、`pyperclip`）。                                                       |
| `README.md`         | 本文件。                                                                                   |

## 📖 使用方法

### 第一步：抓取聊天记录（运行 `微信聊天抓取工具.exe`）

1. **打开微信 PC 版**，进入你想要抓取的聊天窗口（单个联系人或群聊）。

2. **将聊天窗口调整至合适大小，确定起始点、终止点、复制点三点位置坐标**（建议最大化，确保滚动条和“复制”按钮可见）。

3. **打开微信聊天抓取工具-修改脚本中的屏幕坐标参数**（详见下文“配置说明”）。

4. 在终端运行：
   
   ```bash
   python wechat_capture.py
   ```

5. 脚本会倒计时 5 秒，在此期间你可以：
   
   - 将鼠标移至聊天窗口内，**确保窗口处于激活状态**。
   - 如果发现窗口位置不对，可立即关闭终端取消运行。

6. 倒计时结束后，脚本会自动开始：
   
   - 从 `(START_X, START_Y)` 拖拽到 `(END_X, END_Y)`。
   - 点击“复制”按钮（坐标 `(COPY_BTN_X, COPY_BTN_Y)`）。
   - 滚动页面，重复上述过程。

7. **随时按 `Ctrl + Backspace`** 可安全停止抓取，已抓取内容已保存在当前目录下的 `wechat_YYYYMMDD_HHMMSS.txt` 文件中。

> ⚠️ **注意**：抓取过程中，请勿移动鼠标或切换窗口，否则可能影响操作准确性。

---

### 第二步：整理原始文件（运行 `微信聊天整理工具`）

1. 将上一步生成的原始 `.txt` 文件放在与 `wechat_tidy.py` 同一目录下。

2. 在脚本末尾修改 `input_file` 变量为你的原始文件名（默认是 `wechat_yyyymmdd_hhsscc.txt`）。

3. 运行：
   
   ```bash
   python wechat_tidy.py
   ```

4. 程序会生成 `原文件名_tidy.txt` 文件，其中所有消息已去重并按时间顺序（从早到晚）排列。

**你也可以直接调用函数**：

```python
from wechat_tidy import parse_wechat_log
parse_wechat_log('your_raw_file.txt', 'output.txt')
```

## ⚙️ 配置说明（重要！）

由于不同电脑的屏幕分辨率、微信窗口位置不同，原定的参数为1080P下微信全屏的基数参数。**你需要在运行抓取脚本前根据实际情况调整以下坐标参数**（它们位于`wechat_capture.py` 文件顶部）：

| 参数                         | 含义                  | 如何确定                                                        |
| -------------------------- | ------------------- | ----------------------------------------------------------- |
| `START_X`, `START_Y`       | 拖拽**起点**坐标（鼠标按住的位置） | 打开微信聊天窗口，将鼠标移到**最新一条消息的左下角**，记下屏幕坐标（可用微信自带的ALT+A进行坐标点索取。）。  |
| `END_X`, `END_Y`           | 拖拽**终点**坐标（鼠标松开的位置） | 将鼠标移到**聊天框上方正中央。**（基本为微信昵称与消息的中间，可用微信自带的ALT+A进行坐标点索取），记下坐标。 |
| `COPY_BTN_X`, `COPY_BTN_Y` | 复制按钮坐标              | 鼠标移到微信聊天区域右上角的“复制”按钮（或右键菜单中的复制选项，可用微信自带的ALT+A进行坐标点索取），记下坐标。 |
| `SCROLL_STEP`              | 每次滚动像素量             | 默认 `900`，可根据滚动条高度适当调整。                                      |
| `WAIT_AFTER_SCROLL`        | 滚动后等待时间（秒）          | 默认 `3.5`，网络慢或消息加载延迟时可增大。                                    |

> 💡 **小技巧**：在终端中运行 `python -c "import pyautogui; print(pyautogui.position())"` 并移动鼠标，即可实时查看坐标。

## 🛑 安全停止

- **全局热键**：任何时候按下 `Ctrl + Backspace`，脚本会立即终止并保存已抓取内容（适用于任意窗口）。
- **终端中断**：在命令行窗口按 `Ctrl + C` 也可终止，但可能不会保存最后一点数据，建议优先使用全局热键。

## 📁 输出文件

- **原始文件**：`wechat_YYYYMMDD_HHMMSS.txt`，包含所有抓取到的原始内容，每条消息块以 `----------【第X轮消息块】----------` 分隔。
- **整理文件**：`原文件名_tidy.txt`，每一行格式为 `发送者 时间戳 内容`，按时间升序排列，无重复。

## ❓ 常见问题

### Q1：抓取时鼠标乱点，没有选中正确区域？

- **原因**：坐标未适配当前屏幕/窗口大小。
- **解决**：重新测量坐标（建议微信窗口最大化，并确保任务栏未遮挡）。

### Q2：抓取到的内容有很多重复行？

- **原因**：滚动步长 `SCROLL_STEP` 过大，导致两次选区重叠。
- **解决**：减小 `SCROLL_STEP` 值（如设为 `600`），同时增加 `WAIT_AFTER_SCROLL` 确保加载完整。

### Q3：整理后时间排序错乱？

- **可能原因**：原始文件中的时间戳格式有细微差异（如空格数量）。
- **解决**：检查 `wechat_tidy.py` 中的 `parse_time` 函数，确保正则匹配所有可能格式；若格式特殊，可自行调整正则表达式。

### Q4：我能在 macOS 或 Linux 上使用吗？

- 抓取脚本依赖 `pyautogui` 模拟鼠标，理论上可以运行，但微信客户端在不同系统上的界面不同，坐标和按钮位置需重新适配，且全局热键可能不兼容。**建议仅在 Windows 上使用抓取功能**。
- 整理脚本是纯文本处理，可在任何 Python 环境下运行。
